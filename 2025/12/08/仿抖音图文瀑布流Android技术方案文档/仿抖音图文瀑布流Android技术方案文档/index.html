仿抖音图文瀑布流Android技术方案文档

发表于 2025-12-08 更新于 2025-12-25

1. 项目概述

1.1 项目地址

https://github.com/AlexiosChiu/Android-Waterfall-Demo.git

1.2 技术栈

  • 开发语言: Kotlin

  • 架构模式: MVVM

  • 网络请求：：OkHttp

  • 图片加载: Glide

  • 视频播放: ExoPlayer

  • 本地持久化：SharedPreferences

2. 核心功能实现

2.1 底部导航栏

在BaseActivity底部用一个ViewPager2实现，在BasePageViewPagerAdapter中绑定两个Fragment：“首页”HomeFragment和“我”ProfileFragment

2.2 双列瀑布流

主页HomeFragment使用 RecyclerView + StaggeredGridLayoutManager 实现双列瀑布流效果

  1. 双列瀑布流: 2列垂直滚动布局

  2. 间隙处理: 使用 GAP_HANDLING_MOVE_ITEMS_BETWEEN_SPANS策略避免卡片间隙

  3. 封面宽高比限制：FeedAdapter内，设置列宽，限制封面图的宽高比在3:4~4:3之间，绑定视图

  4. 高度缓存: 预计算并缓存每个卡片高度，避免重复计算

2.3 MVVM架构

2.3.1 ViewModel层

HomePageViewModel中处理推文加载/刷新，发射StateFlow （用于加载帖子）/ SharedFlow（用于刷新页面）

2.3.2 UI状态订阅

HomeFragment中设置观察者进行订阅，形成发布-订阅模式

将加载过的推文内容缓存到本地，FeedItem.ImageTextItem对象通过SharedPreferences缓存，图片/视频通过Glide缓存。开屏时，在首次刷新fetch网络数据前先加载历史推文，降低白屏时间，提升了用户体验

2.6.2 图片加载优化

Glide配置优化

包括分配磁盘缓存，增加磁盘缓存线程数，增加网络请求进程数，修改解码器格式，开启内存缓存

缩略图占位

用Glide加载图片时设置先快速加载低分辨率的缩略图（thumbnail）占位，完整图片加载完成后替换，减少白屏时间

优先预加载机制

只接收图片clip时，主页滑动流畅，加载快；自从开始接收视频clip后，主页滑动卡顿严重加剧，推文封面很慢。添加预加载机制，预先加载可见项前后的视频封面

  • 在RecyclerView滚动回调里触发preloadVideoCovers并用lastPreloadRange做防抖，避免重复启动过多任务

  • 用preloadedUrls去重，避免重复请求同一URL

  • 使用独立的CoroutineScope并指定Dispatchers.IO，使预加载尽快在线程池的IO线程上执行

  • 用 Semaphore（videoPreloadSemaphore）限制并发，防止和UI请求互相争抢太多带宽/线程，从而保证每个预加载能完整执行

  • 对Glide请求调用submit后在后台线程上get阻塞等待结果，强制把资源下载/解码并写入Glide的磁盘/内存缓存，后续UI绑定会直接命中缓存

  • Glide仍会根据内部队列调度请求，所以显式设置请求优先级Priority.IMMEDIATE

尽可能减少视频封面解码

多轮优化后，引入视频clip带来的卡顿仍然明显。考虑到在现实场景中，视频封面应当会事先保存在服务器中，而不是让客户端自行解码；再者，用户不会点进每一篇推文查看，更重要的是滑动的流畅度。这里最终选择了尽量用图片clip直接作为封面图，而不去解析第一个视频clip作为封面的方案。从头到尾检索推文中所有clip，用第一个图片clip作为封面图，只有在clips中没有图片只有视频的时候才去解码视频封面。这个优化大大提升了滑动体验

2.6.3 增量更新优化

使用ListAdapter自动计算增量更新，不用自己手动实现差异计算，使用DiffUtil生成最小化刷新操作，并且可实现差异计算与主线程分离

2.6.4 布局优化

尽量减少布局嵌套，特别是减少LinearLayout。在发现主页滑动卡顿后，将原先用多层LinearLayout实现的推文卡片image_text_item_layout.xml布局压平，只用一层ConstraintLayout实现

总体架构逻辑和主页面相似

2.7.1 ViewPager2多媒体横滑容器

包含clips展示，进度条（单个clip不显示），宽高比限制，片段级加载态、失败态

2.7.2 视频播放 (用ExoPlayer实现)

2.7.3 帖子内容

实现标题、正文展示，话题词高亮和话题页跳转，发布时间显示逻辑

实现音乐播放和APP生命周期内静音状态记忆

2.7.4 点赞、关注、分享交互

点赞、关注状态本地持久化，有点击动画和图标改变，点赞状态在主页和推文页间同步。分享键按下后弹出BottomSheetDialog提供分享选项
