3. 重写Item类的方法以实现自定义功能

Item类提供原版物品的各种交互逻辑，自定义物品只需重写即可实现自定义功能。常用的方法有：

  • use -> InteractionResult
    • 右键使用物品时
  • useOn -> InteractionResult
    • 对方块右键使用物品时
  • interactLivingEntity -> InteractionResult
    • 对实体右键使用物品时
  • mineBlock -> boolean
    • 用物品挖掘方块时
  • hurtEnemy -> void
    • 用物品击打实体时

InteractionResult

根据返回InteractionResult的不同成员，客户端中将渲染不同的交互动作。其成员有：

  • SUCCESS ：在客户端做出右键使用的摆手动作
  • SUCCESS_SERVER ：在服务端做出右键使用的摆手动作，但是做给谁看？？
  • CONSUME ：不是吃的意思，并不做出挥手动作，但是会返回ItemContext用于操作
  • FAIL : 表示交互失败，或许能进一步操作
  • PASS ：空定义，常用作不满足使用条件时的返回值
  • TRY_WITH_EMPTY_HAND ：空定义，有什么用？？

使用NeoForge 21.5.42-beta，支持Minecraft 1.21.5

参考：NeoForge文档::Items

本文暂不考虑方块对应物品所需的特殊设置

1.2 定义物品

接下来正式定义物品。写法有三种：

  • 当需要为物品添加性质（堆叠数量、是否抗火等）则在Item.Properties()后链式添加
  • 需要定义在继承的Item子类中重写逻辑的物品时，使用写法1的new 子类名或写法2的子类名::new

1.3 与物品匹配的资源文件

1.3.1 文件目录结构

  • <mod_id>：你的模组ID，例如oceanic_arcana
  • <item_id>：物品的注册名，例如test_item
  • lang/：语言文件目录，用于定义物品的显示名称
  • models/item/：物品模型文件目录，用于定义物品的外观模型
  • textures/item/：物品材质文件目录，用于存放物品的图片资源
  • items/：物品模型映射文件目录，用于定义物品的动态模型映射

1.3.3 模型文件models与物品模型映射

在models/item下需要有与物品同名的json文件定义物品模型样式，以下是MC中最常见的物品模型定义方式：

在textures/item下放置与物品同名的png图片，要求大小为$2^n * 2^n$，常用16*16。若长或宽并不满足，渲染时将向上扩展至某个2的n次幂，向左上对齐

Minecraft NeoForge 开发环境配置 

本文采用环境：NeoForge 21.5.42-beta，支持Minecraft 1.20.5

0. 准备工作

  • JDK21 (Minecraft 1.20及以上版本使用)
  • IntelliJ IDEA
  • NeoForge MDK

1. 导入项目

  • 从NeoForge官网下载定制MDK模板后，用IDEA打开，将自动导入Gradle项目（如果没有，尝试运行gradlew。此时将下载Gradle并反编译Minecraft，国内网络环境需要科学上网
  • 提前在文件-项目结构中将项目SDK修改为21，文件-设置-构建、执行、部署-Gradle中将Gradle JVM修改为项目SDK

2. 修改gradle.properties

  • 修改mod_id、mod_name等基本信息，注意修改mod_group_name后Mod主类要放在相应目录
  • NeoForge已默认使用Parchment映射，只需手动更新parchment_minecraft_version和parchment_mapping_version即可，版本号可在Parchment文档查询
  • 修改完成后，点击右上角刷新Gradle设置

3. 修改Mod主类

  • 修改Mod主类名称，移动到mod_group_name对应的目录中
  • 个人习惯：将字段MODID更名为MOD_ID
  • 修改MOD_ID，注意只能使用小写英文字母和下划线，作为mod的唯一标识命名空间
    • IDEA重命名快捷键：Shift + F6
  • 删除多余的示例代码，可只留下MOD_ID和空的构造函数

4. 尝试运行

  • 运行Gradle任务Client或runClient，查看是否正常
